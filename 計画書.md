# Saru OS / Runtime 技術仕様書（AI実装向け）

## 1. 概要

Saru OS は Linux ベースのゲーム特化プラットフォームである。

初期段階では「独自OS」を目指さず、Linux 上で動作するゲームランタイムおよびゲームランチャーとして開発を行う。

最終的には以下を目標とする：

- 高速起動
- 低レイテンシ
- コントローラ中心 UX
- ゲーム機ライクな固定環境
- 最小構成 Linux ベース OS

---

# 2. 開発フェーズ

## Phase 1（現在）

Ubuntu 等の Linux ディストリビューション上で動作する：

- ゲームランタイム
- ゲームランチャー
- SDK

を開発する。

この段階では通常の Linux アプリケーションとして実装する。

---

## Phase 2

ゲーム専用セッション化。

例：

- 自動ログイン
- ランチャー自動起動
- フルスクリーン化

---

## Phase 3

systemd 依存を削減。

---

## Phase 4

独自 init（PID1）導入。

---

## Phase 5

ゲーム機向け Linux OS 化。

# 4. アーキテクチャ責務

## kernel/

Linux カーネル関連。

責務：

- スケジューリング
- メモリ管理
- GPU ドライバ
- Input subsystem
- ALSA

独自実装しない。

---

## init/

PID1。

責務：

- /proc, /sys, /dev マウント
- デバイス初期化
- GPU 初期化
- オーディオ初期化
- launcher 起動

---

## launcher/

ゲームランチャー。

責務：

- ゲーム一覧 UI
- コントローラ UI
- ゲーム起動
- ゲーム終了監視

描画は SDL3 を使用。

---

## runtime/

ゲーム向け SDK / API。

責務：

- 描画
- 入力
- 音声
- セーブ管理
- 時間管理

---

## sdk/

ゲーム開発者向けヘッダ・ライブラリ。

---

## games/

ゲーム本体。

# 5. プロセスモデル

ランチャーは常駐プロセスとする。

```
keita-init
 └─ launcher
      └─ game process
```

ゲーム起動時：

1. launcher が fork()
2. child process が exec()
3. launcher は waitpid() で待機
4. ゲーム終了後 launcher に戻る

launcher 自体は終了しない。

# 11. ゲームランタイム

## 目的

ゲームが OS と統一的にやり取りできる API を提供する。

以下を抽象化する：

- 描画
- 入力
- 音声
- 時間
- 保存

---

## ディレクトリ構成

```

  ├─ graphics/
  ├─ input/
  ├─ audio/
  ├─ time/
  ├─ storage/
  ├─ system/
  └─ loop/
```

---

void runtime_shutdown();

float runtime_get_delta_time();

    int y
);

void runtime_play_sound(
    const char *path
);

void runtime_save(
    const char *key,
    void *data,
    size_t size
);

    const char *key,
    void *buffer,
    size_t size
